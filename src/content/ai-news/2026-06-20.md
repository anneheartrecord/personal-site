## 今日判断

我今天更关注两类信息：一类是在改真实工作流的人，另一类是在解释 AI 基础设施约束的人。前者里，Claude Code 把 artifact 直接变成团队内可分享的 HTML 页面，这不是一个好看的演示功能，而是在把 agent 输出从聊天记录推进到团队协作对象。Cat Wu 提到他们内部已经拿它做架构变更说明、数据分析和原型，这个细节比官方发布更重要，因为它说明 artifact 不是给个人炫技，而是在替代一部分文档、demo 和状态同步。

我看下来，另一个值得记住的点是 agent 交互正在从 一次性代写 退回到 多轮共写。Nan Yu 说他们早期让 agent 一键生成项目更新，结果用户脑子退出、内容变糊；改成 agent 追问重点、缺失上下文、让用户明确取舍后，质量才上来。这个经验很朴素，但对所有做 agent workflow 的团队都很关键：很多场景里减少摩擦不会带来更好结果，适度 friction 才能把用户判断留在回路里。

基础设施这边，我的判断还是算力没有真正商品化，只是大家前一阵把 GPU 当成了标准云资源来理解。Stephen Balaban 这类一线 infra builder 反复强调的不是卡本身，而是电力、土地、机房壳体、融资能力和把同一块芯片榨出更多吞吐的系统工程。对一线 builder 来说，这意味着未来一年应用层的分水岭，不只是会不会用更强模型，而是谁能围绕成本、延迟、部署边界和人机协作方式，把 AI 真正塞进持续运行的产品里。

[查看原文](https://x.com/_catwu/status/2067674836726694200) · 来源：Cat Wu (@_catwu)

Cat Wu 透露，Claude Team 和 Enterprise 现在可以直接用 Claude Code 部署 HTML 站点，并在团队内分享。她给出的内部使用场景很具体：架构变更说明、数据分析、新原型都已经用 artifact 来沟通。我更看重这里的协作形态变化：agent 输出不再只是会话里的文本，而是能被团队持续查看、迭代、转发的页面对象。我的判断是，这会让 coding agent 从个人提效工具更快走向团队工作流，尤其适合产品评审、工程对齐和轻量 dashboard 这类原本文档很重的环节。

### 2. Nan Yu 复盘：项目更新 agent 从一键代写改回多轮共写

[查看原文](https://x.com/thenanyu/status/2067703108344369306) · 来源：Nan Yu (@thenanyu)

Nan Yu 分享了他们做项目更新 agent 的一次重要修正：早期以为 agent 可以一键生成更新，结果用户直接退出思考，内容变得很空；后来改成多轮交互，agent 主动追问要强调什么、缺什么上下文，产出质量明显提升。我今天很认同这个判断，因为很多团队还在追求全自动，但真实工作流里，更新、总结、决策这类任务往往依赖人的取舍，而不是信息拼接。我的判断是，下一波好用的 agent 不会是一键替你做完，而是把人拉回关键节点，让 steering 成为产品能力。

### 3. Lambda CTO 谈 2026 算力现实：GPU 不是商品，约束在电力和融资

[查看原文](https://www.youtube.com/watch?v=0NttU4CbyVs) · 来源：The MAD Podcast with Matt Turck

这期访谈里，Lambda 的 Stephen Balaban 讲得最有价值的不是行情，而是 AI 算力为什么始终没有变成真正商品化资源：H100 价格指数会误导判断，真正的瓶颈常常是土地、电力、机房壳体、融资能力，以及如何从同一批芯片里榨出更多训练和推理价值。我看下来，这比单纯讨论 GPU 供需更有用，因为应用层团队最终都会被延迟、成本和可获得性反向塑形。我的判断是，未来一段时间 infra 竞争优势会继续来自系统工程和资本组织能力，而不是简单转售卡时。

### 4. Nikunj 用 Claude Code 和 Codex 做 SEO 实验，拿到 16k 展现

[查看原文](https://x.com/nikunj/status/2067830061009633294) · 来源：Nikunj Kothari (@nikunj)

Nikunj 说他悄悄上线了一个 side project，核心实验不是做新功能，而是让 Claude Code 和 Codex 把网站的 SEO 和 GEO 优化做到极致。结果是在没有外链、没有社媒导流、没有 Reddit 操作的情况下，28 天拿到约 1.6 万展现和 94 次点击。这个数字不算爆炸，但方向很实用：agent 已经开始进入增长侧的重复劳动，而且结果可以被搜索流量直接验证。我的判断是，接下来很多小团队会把 coding agent 当成增长运营的执行层，而不只是工程助手。
