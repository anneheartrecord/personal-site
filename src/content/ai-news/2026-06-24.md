## 今日判断

我今天看下来，真正值得一线 builder 记的，不是又有谁喊出了更大的模型口号，而是几条很具体的落地信号：评测开始从研究话题变成产品能力，HTML 重新成为 agent 交付物，设计稿到部署链路继续缩短。这里面最有工程味道的是 Aaron Levie 讲 evals 那条，因为他说的不是抽象方法论，而是企业最终能不能把 agent 接进真实流程，取决于你有没有能力定义任务、验收结果、持续迭代基线。这会直接决定团队接下来该投在 prompt、模型切换，还是投在 workflow instrumentation 上。

另一条我更在意的是 HTML 正在回潮，而且不是复古意义上的网页，而是 agent 最容易生成、最容易审阅、也最容易分享的中间产物。Box 说开始支持预览和安全分享 HTML，Vercel 说 Claude Design 可以一键到部署，这两条放在一起看，比单条产品更新更有意思：agent 先产出可运行界面，再进入版本管理、托管、协作，已经开始形成闭环。对 builder 来说，这意味着很多过去要先进设计系统、再进前端工程的流程，会被一段可运行 HTML 先行替代。

我的判断是，这一轮不是大家比谁先接上最新模型，而是谁更快把评测、可运行产物和部署链路连起来。会做 demo 已经不够，下一步会拉开差距的是：你能不能把 agent 产出的页面、脚本、修复建议直接放进团队真实工作流里，并且知道它什么时候好用、什么时候该拦住。

### 1. Aaron Levie：企业里真正决定 agent 成败的是 evals

[查看原文](https://x.com/levie/status/2069228335255949775) · 来源：Aaron Levie (@levie)

Aaron Levie 公开把最近模型、agent 和企业自动化的进展都归到同一个瓶颈：evals。他的意思很明确，企业未来的核心能力不是会不会接模型 API，而是能不能理解自己的工作流，并持续判断 agent 在每个环节到底做得怎么样。我看这条重要，是因为它把很多团队含糊讨论的“AI 落地”翻成了工程问题：任务定义、验收标准、失败样本、基线追踪。我的判断是，接下来企业 agent 项目最容易拉开差距的，不是 prompt 技巧，而是谁先把内部 eval 管线搭起来。

### 2. Zara Zhang 把 Claude Code 做前端幻灯片的技能链路讲透了

[查看原文](https://x.com/zarazhangrui/status/2069311440692072481) · 来源：Zara Zhang (@zarazhangrui)

Zara Zhang 发布了一个 11 分钟 walkthrough，拆解她那个拿到 2.2 万星的 Frontend Slides skill 是怎么用 Claude Code 生成 HTML 幻灯片的，也讲了如何做自己的 skill、怎么插入图片视频、怎么发布。这类内容比模型跑分更有用，因为它给的是可复制的工作流：把一次成功的 prompt 和文件组织方式沉淀成 skill，再让别人复用。我看这条的价值在于，它证明 coding agent 的门槛已经从“会不会用”转向“会不会包装成团队资产”。我的判断是，skill 化会成为下一阶段 agent 使用效率的主要杠杆。

### 3. OpenAI 发布 GPT-5.5-Cyber，并把 Codex 安全能力推向补丁场景

[查看原文](https://x.com/sama/status/2069121360744550796) · 来源：Sam Altman (@sama)

Sam Altman 宣布 GPT-5.5-Cyber 全量可用，并把 Patch The Planet 和 Codex Security 一起推出来，强调目标不是只发现漏洞，而是帮助企业真正修补问题。信息本身还偏发布口径，没有展开架构、定价或真实修复流程，但方向很清楚：安全 agent 正在从审计助手转向 remediation worker。我会先把它当成一个产品信号，而不是马上下结论的能力飞跃。我的判断是，安全场景会比通用办公更早要求 agent 从分析走向执行，因为补丁、配置和修复本来就是更结构化的工作。

### 4. Box 开始把 HTML 当成 agent 产物来管理和分享

[查看原文](https://x.com/levie/status/2069140445205348432) · 来源：Aaron Levie (@levie)

Aaron Levie 说，Box 现在支持预览、编辑、版本管理和安全分享 HTML 内容，明确点名这对 agent 生成的内容很有用。这个变化看起来小，但我觉得它非常贴近真实工作流：很多 agent 最自然的输出不是 PDF 或截图，而是一份可运行的 HTML 原型、报告面板或小工具。如果协作系统不能直接接住这类产物，团队就只能继续截图传来传去。我的判断是，HTML 会成为 agent 时代最重要的中间格式之一，谁能把它接入权限、审阅和版本体系，谁就更接近真正的企业落地。

### 5. Vercel 把 Claude Design 到上线之间再砍掉一步

[查看原文](https://x.com/rauchg/status/2069219190834127276) · 来源：Guillermo Rauch (@rauchg)

Guillermo Rauch 用一句话宣布 Claude Design 可以一键部署到 Vercel。单看这条信息量不大，但放在现在的产品链路里，它代表设计生成、前端产物和托管平台之间的摩擦继续下降。以前团队最头疼的是 demo 很快出来，但从 demo 到可分享链接、到真实环境部署之间还有很多手工步骤。现在这些步骤被平台吃掉之后，agent 产出的价值会更容易被团队内部验证。我看这条不是模型新闻，而是工作流新闻。我的判断是，接下来最强的 AI 前端体验会来自生成工具和托管平台的深度联动。

### 6. Mike Krieger 讨论 Fable 5：模型价值要看几周后的真实用法

[查看原文](https://www.youtube.com/playlist?list=PLuMcoKK9mKgHtW_o9h5sGO2vXrffKHwJL) · 来源：AI & I by Every

这期对谈里，Mike Krieger 讲的重点不是 Fable 5 首日多惊艳，而是一个更接地气的判断：模型到底改变什么，通常要几周高强度使用之后才看得清。他提到很多人刚上手时其实没有把模型推到极限，后面才会重新理解它能接管哪些任务。我会选这条，是因为它提醒 builder 不要被 day-one 叙事带着跑。真实世界里，模型能力不是发布会当天决定的，而是你有没有把它嵌进每天的写作、编码、协作里。我的判断是，未来对模型的最好评测会越来越像使用复盘，而不是 benchmark 截图。
