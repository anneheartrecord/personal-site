## 今日判断

我今天更关注的不是又一个模型发布会，而是几条真正把模型放进工作流里的人在说什么。看下来，大家关心的已经不是单点 benchmark，而是 agent 在真实环境里的三个老问题：工具调用稳不稳、外部系统接得顺不顺、长任务到底会不会半路放弃。Josh 把 Gemini 用户前 10 个反馈直接摊开，第一名是 Google Workspace 集成可靠性，第二名是 tool calling，第四名是 MCP 和 custom skills。这个排序很说明问题：买家现在要的不是更会聊天，而是能接企业系统、能稳定执行、能组织长期工作。

### 1. Gemini 产品负责人公开用户前十反馈，重点全在可靠性和集成

[查看原文](https://x.com/joshwoodward/status/2075241749048401936) · 来源：Josh Woodward (@joshwoodward)

Josh Woodward 把过去 12 小时 1400 多条 Gemini 用户回复做了 stack rank，排在最前面的不是新模型，而是 Google Workspace 集成更可靠、tool calling 更稳定、聊天项目组织更好，以及 MCP 和 custom skills 支持。对一线 builder 来说，这比任何发布会口号都更有用，因为它把 agent 真正卡住落地的地方讲明白了：不是能力缺 5 分，而是系统接口和执行稳定性不够。我看这条的意义在于，下一阶段竞争会更像产品工程战，不是谁参数更多。

### 2. Box 用企业文档任务评测 GPT-5.6，提升集中在复杂推理场景

[查看原文](https://x.com/levie/status/2075287443411222628) · 来源：Aaron Levie (@levie)

Aaron Levie 说 Box 用自家的 Complex Work eval 测了 GPT-5.6 Sol，在金融和医疗等高难企业文档任务上，相比 GPT-5.5 有明显提升，尤其体现在多步骤推理里少犯前置假设错误，比如财务模型起始日期锚定、医疗案例诊断和干预判断。这条重要，不是因为又一个分数涨了，而是它说明更强的模型价值主要出现在错误会层层传导的任务里。我判断企业今年最愿意为模型升级买单的，也正是这种错一次就要重做整条链路的工作。

<a id="20260711-03"></a>

### 3. 一线使用者对 GPT-5.6 的体感：更能坚持做完任务，但更烧 token

[查看原文](https://x.com/petergyang/status/2075345016437039600) · 来源：Peter Yang (@petergyang)

Peter Yang 在实际试了一天 GPT-5.6 后给出的判断很直接：它的优点不是更会说，而是遇到复杂任务时更不轻易放弃，配合 browser use、图像、语音和插件，已经更像能处理白领工作的协作对象；代价是 token 消耗看起来比 GPT-5.5 更高。对 builder 来说，这条提醒很实用，agent 的体验提升很多时候来自更强的任务坚持性和工具联动，而不是单轮回答更漂亮。我今天的判断是，如果你做的是高价值长任务，贵一点可能值得；如果是高频低价流量，成本会立刻变成第一约束。

<a id="20260711-04"></a>

### 4. Replit 观察到一个反直觉趋势：AI 让写代码更松，运行时反而更硬

[查看原文](https://x.com/amasad/status/2075423115052790054) · 来源：Amjad Masad (@amasad)

Amjad Masad 说，AI 让编码过程变得没那么僵硬，但他们的 infra 团队反而第一次开始大量写 formal specs，系统也在变得更 deterministic、更 resilient。原因很简单：你越想让模型和 agent 快速产出，底下的 runtime、部署链路和隔离边界就越不能模糊。这条我很认同，因为很多团队现在把注意力全放在生成速度，忽略了执行面。我的判断是，未来真正拉开差距的，不只是前台 agent 有多聪明，而是后台系统能不能把不确定输入收敛成可控结果。

### 5. 模型开始自带工具偏好，Swyx 点出邮件服务选择里的隐性分发

[查看原文](https://x.com/swyx/status/2075376938676621752) · 来源：Swyx (@swyx)

Swyx 提到一个很具体的现象：即使他已经配好现有 transactional infrastructure，多个前沿模型在发邮件相关任务里还是倾向调用 Resend。这不是一句玩笑那么简单，它暴露了 agent 生态里新的分发入口正在形成——不是搜索排名，而是模型在工具选择时的默认先验。对做开发者工具和 SaaS 的团队，这件事很重要，因为你未来争的未必只是开发者心智，还包括能不能进入模型默认工作流。我看这会催生一轮面向 agent 的集成优化和提示工程式渠道竞争。
